# Claude Code Rules for Darwin UI

## Execution Rules
- Do NOT stop to ask for permissions - execute immediately
- Always use sub-agents to parallelize work
- Work like a full-stack team of senior developers
- Put maximum effort into every task
- Be thorough - search and verify everything

## Design Rules
- Follow Apple's modern design language strictly
- NO gradients on icons, buttons, or UI elements
- Use Lucide icons exclusively
- Keep animations smooth and Apple-like
- Traffic lights must be proper dots
- Dock behavior must match macOS exactly

## Code Quality
- Fix all broken functionality before adding new features
- Test every component interaction
- Ensure consistent styling across all components
